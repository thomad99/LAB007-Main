007Trade · Paper trader — Broker & storage

API keys stay in environment variables on the host (e.g. Render). Watchlists, trade plans, and your “context for briefs” text are stored in a JSON file on disk — see below.

Alpaca (supported)

Use Alpaca for paper trading first (ALPACA_PAPER=true, default).

  • ALPACA_API_KEY_ID — API key ID
  • ALPACA_API_SECRET_KEY — secret
  • ALPACA_PAPER — true (paper-api) or false (live trading — extra caution)

Endpoints: paper https://paper-api.alpaca.markets, live https://api.alpaca.markets.

Claude (strategy briefs)

Uses the same Anthropic key as other LAB007 features:

ANTHROPIC_API_KEY

Optional: TRADE007_ANTHROPIC_MODEL (defaults to Claude Sonnet).

Persistent dashboard state

Watchlists, trade plans, strategy selection, and context notes are written to a single state file on the server so they survive restarts when the disk path is on a mounted volume.

  • LAB007_DATA_DIR — set this to your Render persistent disk mount (e.g. /var/data/lab007) so state lives under …/Trade007/state.json.
  • TRADE007_DATA_PATH — optional full path to state.json if you want an explicit file location.
